# 文案风格规范（用户化 + 专业化）

## 目标
- 让用户一眼看懂当前状态、下一步操作、失败原因。
- 保持专业、克制、可执行，不用内部黑话和隐喻。

## 语气规则
- 直接描述系统行为：`正在生成候选方案`，不要 `正在构建平行宇宙`。
- 用用户视角命名：`历史项目`、`模型配置`，不要 `档案库`、`导演包`（除非是专业术语且有解释）。
- 同一概念统一叫法：
  - `候选方案`（不要与 `平行宇宙` 混用）
  - `一致性锁定`（不要与 `母版锁定`、`锁死` 混用）
  - `图像模型`（不要与 `生图模型` 混用）

## 状态文案模板
- 进行中：`正在{动作}`，例如 `正在准备提示词...`
- 成功：`{动作}完成`，例如 `追加方案完成。`
- 失败：`{动作}失败: {原因}`，例如 `方案生成失败: 请求超时`
- 空状态：`当前无任务，等待开始。`

## 按钮文案模板
- 主按钮用动词开头：`开始生成`、`确认方案并开始生成`
- 次按钮清晰区分：`深度生成（12 方案）`
- 危险操作明确：`删除记录`

## 禁用词（尽量避免）
- 情绪化或口语化：`活着`、`锁死`、`废片`、`脑波`
- 内部隐喻：`暗房`、`平行宇宙`（面向普通用户界面）

## 诊断提示规范
- 先说现象，再给动作：
  - `网关状态异常，请检查后端模型配置`
  - `触发限流，请稍后重试`
